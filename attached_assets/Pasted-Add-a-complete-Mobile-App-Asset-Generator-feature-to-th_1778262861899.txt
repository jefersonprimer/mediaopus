Add a complete "Mobile App Asset Generator" feature to the existing Image Toolkit application.

The feature must allow users to upload a single image/logo and automatically generate all required mobile app assets for Android, iOS, Expo, and Progressive Web Apps.

All image generation and processing must run entirely client-side in the browser without any backend, APIs, cloud processing, authentication, or external services.

Core Requirements:

* Upload a single source image/logo
* Automatically generate mobile app assets in correct sizes and formats
* Generate transparent PNG outputs when necessary
* Preserve high image quality
* Keep sharp edges and anti-aliasing
* Support logos, icons, illustrations, and flat graphics
* Fully frontend-only processing
* Fully static deployment compatible
* No backend server
* No external APIs
* No paid services
* No authentication

Generate the following assets automatically:

Android:

* android-icon-background.png (512x512)
* android-icon-foreground.png (512x512)
* android-icon-monochrome.png (432x432)
* adaptive-icon.png
* notification-icon.png

Expo React Native:

* icon.png (1024x1024)
* adaptive-icon.png
* splash-icon.png

iOS:

* app-store-icon.png (1024x1024)
* apple-touch-icon.png

Web/PWA:

* favicon.png
* favicon-16x16.png
* favicon-32x32.png
* favicon-48x48.png
* web-app-manifest icons

Feature Requirements:

* Automatically center and scale uploaded artwork
* Configurable foreground scale percentage
* Default foreground scaling between 50% and 66%
* Maintain safe padding zones for Android adaptive icons
* Support transparent backgrounds
* Generate monochrome icons automatically
* Allow custom background colors
* Allow gradient backgrounds
* Preview all generated assets before download
* Generate ZIP file containing all assets
* Preserve original aspect ratio
* Support drag-and-drop uploads
* Mobile-friendly responsive UI
* Dark mode support
* Smooth modern animations and transitions

Technical Requirements:

* Use Canvas API for image generation
* Use TypeScript everywhere
* Use reusable utility functions
* Use reusable hooks
* Use efficient client-side image processing
* Properly clean up object URLs and canvases
* Strong typing and scalable architecture
* Optimized performance
* Batch export support

UI Additions:

* "Mobile Asset Generator" section/page
* Live preview grid for generated assets
* Foreground scale slider
* Background color picker
* Monochrome preview toggle
* Platform presets:

  * Expo
  * Android Native
  * iOS
  * PWA
* Download individual assets
* Download complete ZIP package
* Modern SaaS-style interface

Advanced Features:

* Automatically generate monochrome Android icons
* Automatically generate adaptive icon safe zones
* Allow exporting ready-to-use Expo configuration snippets
* Allow exporting app.json icon configuration examples
* Optional background removal integration before asset generation

Suggested Processing Logic:

* Resize and center artwork into target canvas sizes
* Apply configurable scaling percentage
* Maintain transparent padding/safe areas
* Generate monochrome versions by converting artwork to solid black or white
* Generate multiple resolutions automatically
* Package generated assets into ZIP downloads

The implementation should be production-ready, scalable, lightweight, reusable, and integrated cleanly into the existing Image Toolkit architecture.
