   Current behavior is incorrect.

The current implementation simply scales the entire uploaded image into a transparent canvas. This is NOT the expected behavior.

Correct behavior:

* Automatically remove the image background first
* Detect and remove white, black, or near-solid backgrounds
* Preserve only the foreground artwork/logo
* Export transparent PNG
* Keep the artwork centered
* Preserve transparency
* Maintain sharp edges and anti-aliasing
* Output must be exactly 512x512
* Do NOT place the original image inside another transparent image without processing
* The artwork should naturally occupy the safe area without unnecessary padding

The same correction must also be applied to:

* splash-icon.png (1024x1024)

3. android-icon-monochrome.png
   Current behavior is incorrect.

Correct behavior:

* Remove background completely
* Preserve transparency
* Convert the foreground artwork into a fully monochrome version
* Convert all visible pixels to solid black or solid white
* Preserve anti-aliased edges using alpha transparency
* Do NOT preserve original colors
* Do NOT flatten transparency
* Output must be exactly 432x432
* Result should follow Android monochrome adaptive icon guidelines

Implementation Requirements:

* Use Canvas API pixel manipulation

* Preserve alpha transparency correctly
