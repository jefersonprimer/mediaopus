Enhance the existing Image Toolkit application by adding a non-AI solid color background removal feature optimized for logos, icons, illustrations, screenshots, and flat graphics.

This feature must work entirely client-side in the browser without any backend, API calls, cloud processing, authentication, or external services.

The goal is to provide a fast and lightweight alternative to AI background removal for images with solid or near-solid backgrounds such as white, black, or custom colors.

Requirements:

* Add a "Remove Solid Background" feature
* Detect and remove white backgrounds accurately
* Support removing black backgrounds
* Support removing custom colors with a color picker
* Generate transparent PNG outputs
* Preserve smooth anti-aliased edges
* Avoid damaging logo borders and thin details
* Work well for logos, icons, screenshots, and flat assets
* Process images instantly without AI models
* Fully frontend-only processing
* Support batch processing for multiple images
* Allow downloading processed images individually
* Allow ZIP download for batch exports
* Mobile-friendly responsive UI
* Dark mode support
* Keep the existing modern SaaS design language

Technical Requirements:

* Use Canvas API and pixel manipulation
* No AI models
* No backend
* No APIs
* No external processing
* No paid services
* No authentication
* Fully static deployment compatible
* Use TypeScript
* Strong typing and reusable architecture
* Optimized performance for large images
* Proper memory cleanup for object URLs and canvases

Image Processing Requirements:

* Remove pixels close to the selected background color
* Add configurable threshold/sensitivity controls
* Preserve transparency edges smoothly
* Support anti-aliasing preservation
* Prevent jagged edges on logos
* Allow live preview before export
* Maintain original image resolution

UI Additions:

* "Remove White Background" quick action
* "Remove Black Background" quick action
* Custom color picker tool
* Threshold/sensitivity slider
* Before/after comparison preview
* Transparent checkerboard preview background
* Download transparent PNG button
* Batch processing controls
* Smooth transitions and loading animations

Implementation Details:

* Create reusable image processing hooks
* Create utility functions for pixel-level color removal
* Use RGBA pixel analysis via Canvas API
* Convert matching pixels to transparent alpha
* Use efficient loops for performance optimization
* Structure code cleanly for future expansion

Suggested Algorithm:

* Read image pixels using CanvasRenderingContext2D
* Compare RGB values against target background color
* If pixel color is within threshold range:

  * Set alpha channel to 0 (transparent)
* Preserve partially transparent edge pixels for smooth results

The implementation should be production-ready, scalable, lightweight, and integrated cleanly into the existing project architecture alongside the AI background removal feature.
